Multilotto网站改版开发规范

  • dev开发环境地址dev.multilotto.com
  • git仓库地址

网站结构

  • app 自定义的第三方类，比如支付类，广告商，casino游戏商
  • class 网站自定义类
  • controllers 网站控制类
  • cron PHP cron脚本
  • env 环境变量
  • files 日志和翻译缓存文件地址
  • inc 全局函数和配置文件
  • public 公共入口
    • assets 公共静态文件入口
    • payment 对外（第三方支付）接口
    • services 对外（第三方服务商）接口
    • sitemaps 网站地图
    • .htaccess 网站apache配置文件
    • index.php 主入口
  • themes 主题文件
    • resources 主题静态资源文件
    • templates 主题模板文件
    • theme.json 主题模板配置文件
  • vendor 框架第三方类库

改版规范

  • 新建主题文件mmlotto
  • 静态资源文件单独放在mmlotto主题下面
  • 公用js统计放在main.min.js下面
  • 每个页面如果需要用到js单独创建js文件，单独加载。JS可以公用的需使用公用JS。
  • 使用现有架构实现前后端分离，业务无关的数据可以通过PHP直接加载，业务相关的判断数据需要异步加载（暂）
  • HTML中所有JS用到的id和class需要使用js_开头， 比如：class="js_loginform"
  • 新增路由配置放在最后，创建新的Controller类：mm.auth.controller.php Class MMAuthController

Routing路由

  • 路由配置
<?php
$router->get('/lotto', ['c' => 'games', 'a' => 'getIndex'], 'lotto'); 
$router->post('/lotto', ['c' => 'games', 'a' => 'getIndex'], 'lotto'); 
// 三个参数 route路由, parameters入参, name别名
$router->map('/lotto-statistics-numbers', ['c' => 'statistics', 'a' => 'getIndex'], 'GET|POST', 'stats');
// map可以同时适配post和get
$router->get('/lotterynews/[i:offset]?', ['c' => 'news', 'a' => 'getIndex'], 'news');
// url占位符, I:offset表示offset入参格式为数字， 入参格式如下：
private $matchTypes = [
    'i'  => '[0-9]++',
    'a'  => '[0-9A-Za-z]++',
    'h'  => '[0-9A-Fa-f]++',
    '*'  => '.+?',
    '**' => '.++',
    ''   => '[^/]++'
];
?>
