最佳原则

  • 坚持制定好的代码规范。
  • 无论团队人数多少，代码应该同出一门。
  • 如果你想要为这个规范做贡献或觉得有不合理的地方，可以提出讨论。

命名规则

项目命名

  • 全部采用小写方式， 以下划线分隔。例：my_project_name

目录命名

  • 参照项目命名规则，有复数结构时，要采用复数命名法。例：scripts, styles, images, data_models

PHP文件命名

  • 参照项目命名规则。例：retina_sprites.php

HTML文件命名

  • 参照项目命名规则。例：error_report.html

JS文件命名

  • 参照项目命名规则。例：account_model.js

PHP

语法

  • 缩进使用soft tab（4个空格）；
  • 静态页面使用PHP标签必须使用完整标签，比如：<?php echo $val['datamoney']; ?>。
  • if else swith for while等语句大括号直接跟在控制语句之后，不换行。

变量

  • 普通变量：所有字母都使用小写；对于一个变量使用多个单词的，使用'_'作为每个词的间隔。 例如：$base_dir、$red_rose_price等
  • 静态变量：静态变量使用小写的s_开头；静态变量所有字母都使用小写；多个单词组成的变量名使用'_'作为每个词的间隔。例如：$s_base_dir、$s_red_rose_prise等。
  • 局部变量：所有字母使用小写；变量使用''开头；多个单词组成的局部变量名使用''作为每个词间的间隔。例如：$_base_dir、$_red_rose_price等。
  • 全局变量：全局变量应该带前缀'g'。例如：global $gLOG_LEVEL; global $gLOG_PATH;等。
  • 全局常量：所有字母使用大写；全局变量多个单词间使用’_'作为间隔。例如：$BASE_DIR、$RED_ROSE_PRICE等。

函数

  • 必须有函数名，入参，出参注释。
  • 采用驼峰命名法，第一个单词字母小写，后续单词字母大写。
  • 无论是函数声明还是函数表达式，'('前不要空格。
  • 函数'}'后,'{'前要加空格。
  • 函数调用括号前不需要空格。
  • 参数之间用', '分隔，注意逗号后有一个空格。
  • 大括号在直接跟在控制语句之后不换行 
<?php
    function toString($a, $b) {
        // do something
    };
    // good
    toString($a, $b);
    if($a == 1) {
        ...
    } else {
        ...
    }
?>

类命名

  • 采用驼峰命名法，所有单词字母大写

数组

  • 数组属性名使用双引号；
  • 数组以缩进的形式书写，不要写在一行；
  • 数组最后不需要有逗号。
<?php
    // good
    $arr = array(
        "a" => "1",
        "b" => 2
    );
?>

括号

  • 下列关键字后必须有大括号（即使代码块的内容只有一行）：if, else, for, while, do, switch, try, catch, finally, with。
<?php
    // good
    if(condition) {
        doSomething();
    }  
?>

HTML

语法

  • 缩进使用soft tab（4个空格）；
  • 嵌套的节点应该缩进；
  • 在属性上，使用双引号，不要使用单引号；
  • 属性名全小写，用中划线做分隔符；
  • 不要在自动闭合标签结尾处使用斜线（HTML5 规范 指出他们是可选的）；
  • 不要忽略可选的关闭标签，例：
    • 和 。 
<!DOCTYPE html>
<html>
    <head>
        <title>Page title</title>
    </head>
    <body>
        <img src="images/company_logo.png" alt="Company">
        <h1 class="hello-world">Hello, world!</h1>
    </body>
</html>

HTML5页面规范

  • DOCTYPE启用网页标准模式，每个页面规范原则上使用一套模板。
  • 每个页面编码使用"UTF-8", 包括文件编码也使用"UTF-8"。
  • 根据HTML5规范, 通常在引入CSS和JS时不需要指明 type，因为 text/css 和 text/javascript 分别是他们的默认值。
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8">
        <!-- External CSS -->
        <link rel="stylesheet" href="code_guide.css">
        <!-- External JS -->
        <script src="code_guide.js"></script>
    </head>
    <!-- In-document CSS -->
    <style>
        ...
    </style>
    <!-- In-document JS -->
    <script>
        ...
    </script>
</html>

属性顺序

  • 属性应该按照特定的顺序出现以保证易读性；
  • class
  • id
  • name
  • data-*
  • src, for, type, href, value , max-length, max, min, pattern
  • placeholder, title, alt
  • aria-, role
  • required, readonly, disabled
<a class="..." id="..." data-modal="toggle" href="#">Example link</a>
<input class="form-control" type="text">
<img src="..." alt="...">

JS不生成标签

  • 为了便于管理HTML标签，所有需要JS中生成HTML标签的需要在当前HTML页面写好模板，JS只负责替换模板数据，也就是说JS不不做生成标签的操作。

JavaScript

  • 不要混用tab和space；
  • 对上下文this的引用只能使用'_this', 'that', 'self'其中一个来命名；
  • switch的falling through和no default的情况一定要有注释特别说明；
  • 不允许有空的代码块。

缩进

  • 使用soft tab（4个空格）。

分号

以下几种情况后需加分号：

  • 变量声明
  • 表达式
  • return
  • throw
  • break
  • continue
  • do-while
/* var declaration */
var x = 1;

/* expression statement */
x++;

/* do-while */
do {
    x++;
} while (x < 10);

空格

以下几种情况不需要空格：

  • 对象的属性名后
  • 前缀一元运算符后
  • 后缀一元运算符前
  • 函数调用括号前
  • 无论是函数声明还是函数表达式，'('前不要空格
  • 数组的'['后和']'前
  • 对象的'{'后和'}'前

以下几种情况需要空格：

  • 二元运算符前后
  • 三元运算符'?:'前后
  • 单行注释'//'后（若单行注释和代码同行，则'//'前也需要），多行注释'*'后
  • 对象的属性值前
  • for循环，分号后留有一个空格，前置条件如果有多个，逗号后留一个空格
  • 函数的参数之间
  • 函数'}'后,'{'前要加空格。
// good
var a = {
    b: 1
};
// good
++x;
y++;
z = x ? 1 : 2;
// good
var a = [1, 2];
// good
var a = (1 + 2) * 3;
// good
for (var i = 0; i < 6; i++) {
    x++;
}

单行注释

  • 双斜线后，必须跟一个空格；
  • 缩进与下一行代码保持一致；
  • 可位于一个代码行的末尾，与代码间隔一个空格。
if (condition) {
    // if you made it here, then all security checks passed
    allowed();
}
var zhangsan = 'zhangsan'; // one space after code

多行注释

  • 最少两行, '*'后跟一个空格，具体参照右边的写法；
  • 建议以下情况下使用：
  • 难于理解的代码片段。
  • 可能存在错误的代码段。
  • 业务逻辑强相关的代码。
/* 
 one space after '*'
 one space after '*'
 */

文档注释

  • 各类标签@param, @method等;
  • 建议在以下情况下使用：
  • 所有常量
  • 所有函数
  • 所有类
/**
 * @func
 * @desc 一个带参数的函数
 * @param {string} a - 参数a
 * @param {number} b=1 - 参数b默认值为1
 * @param {string} c=1 - 参数c有两种支持的取值</br>1—表示x</br>2—表示xx
 * @param {object} d - 参数d为一个对象
 * @param {string} d.e - 参数d的e属性
 * @param {string} d.f - 参数d的f属性
 * @param {object[]} g - 参数g为一个对象数组
 * @param {string} g.h - 参数g数组中一项的h属性
 * @param {string} g.i - 参数g数组中一项的i属性
 * @param {string} [j] - 参数j是一个可选参数
 */
function foo(a, b, c, d, g, j) {
    ...
}

引号

  • 最外层统一使用单引号。
// good
var y = 'foo',
    z = '<div id="test"></div>';

变量命名

  • 标准变量采用驼峰式命名。
  • 'ID'在变量名中全大写。
  • 'URL'在变量名中全大写。
  • 'Android'在变量名中大写第一个字母。
  • 'iOS'在变量名中小写第一个，大写后两个字母。
  • 常量全大写，用下划线连接。
  • 构造函数，大写第一个字母。
  • jquery对象必须以'$'开头命名。
/* 例如： */
var thisIsMyName;
var goodID;
var reportURL;
var AndroidVersion;
var iOSVersion;
var MAX_COUNT = 10;
function Person(name) {
    this.name = name;
}
// good
var $body = $('body');

变量声明

  • 一个函数作用域中所有的变量声明尽量提到函数首部，用一个var声明，不允许出现两个连续的var声明。
function doSomethingWithItems(items){
    // use one var
    var value = 10,
        result = value + 10,
        i,
        len;
    for (i = 0, len = items.length; i < len; i++) {
        result += 10;
    }
}

函数

  • 无论是函数声明还是函数表达式，'('前和')'后不需要空格；
  • 函数调用括号前不需要空格；
  • 立即执行函数外必须包一层括号；
  • 参数之间用', '分隔，注意逗号后有一个空格。
// no space before '(', but one space before'{'
var doSomething = function(item){
    // do something
};
function doSomething(item){
    // do something
}
// good
doSomething(item);
// requires parentheses around immediately invoked function expressions
(function(){
    return 1;
})();
// good
[1, 2].forEach(function(){
    ...
});
// use ', ' between function parameters
var doSomething = function(a, b, c){
    // do something
};

数组、对象

  • 对象属性名不需要加引号；
  • 对象以缩进的形式书写，不要写在一行；
  • 数组、对象最后不要有逗号。
// good
var a = {
    b: 1,
    c: 2
};

括号

  • 下列关键字后必须有大括号（即使代码块的内容只有一行）：if, else, for, while, do, switch, try, catch, finally, with。
// good
if (condition) {
    doSomething();
}
